## Free Tier Design {#free-tier}

### The Critical Question: What to Limit?

**Problem:** Auto-reload sends hundreds of bundles per session
- Counting individual bundles = punishing the main feature
- Time-based trials = Standard but imprecise
- Session-based = Fair but complex

### Recommended Free Tier: Time-Based Trial

```
AppTuner Free (14-day trial):
âœ… All features unlocked
âœ… Unlimited bundles
âœ… Unlimited devices
âœ… Console logging
âœ… Error overlay
âœ… Hot reload
âœ… Everything works!

After 14 days:
â†’ "Trial ended. Unlock lifetime access for $49"
```

**Why this is best:**
1. **Simple to understand:** "Free for 2 weeks, then $49 forever"
2. **No bundle counting:** Users can iterate freely
3. **Fair evaluation:** 14 days is enough to build a real project
4. **Industry standard:** Matches competitor trials
5. **High conversion:** Users get hooked in 2 weeks

### Alternative: Session-Based (Advanced)

```
AppTuner Free:
âœ… First 30 development sessions
âœ… Unlimited bundles per session
âœ… Session = connect to disconnect (or 1-hour idle timeout)

After 30 sessions:
â†’ "You've completed 30 sessions. Unlock unlimited for $49"
```

**Why session-based:**
- Prevents abuse (can't use free forever by disconnecting)
- Fair for part-time developers (30 sessions = 2-4 weeks of work)
- Encourages focused work sessions
- More complex to implement

**Recommended: Start with 14-day trial, iterate to sessions if needed**
